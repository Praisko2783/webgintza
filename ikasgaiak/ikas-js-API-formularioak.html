Formularioaren prozesua    

Formularioen APIa propietate, metodo eta gertaera multzo bat da, inprimakiak prozesatzea eta gure baliozkotze-sistema sortzea. APIa formularioak eta formularioen elementuak; beraz, gertaerei erantzun diezaiekegu edo metodoak elementu berberetatik. Hona hemen metodo batzuk

elementuarekin erabiltzeko:

  • submit () — Metodo honek formularioa bidaltzen du.
  • reset() — Metodo honek formularioa berrabiarazten du (lehenetsitako balioak elementuak).
  • checkValidity () — Metodo honek balio boolearra itzultzen du, formularioa baliozkoa den ala ez adierazteko.

APIak hurrengo ekitaldia ere eskaintzen du, txertatzen den bakoitzean karakterea edo balio bat aukeratzen da formularioko elementu batean.

input — Gertaera hau formularioan edo haren elementuetan hasten da, erabiltzaileak karaktere bat txertatzen edo ezabatzen du sarrera-eremu batean, baita balio berria.

change — Ekitaldi hau formularioan edo haren elementuetan hasten da, balio berri bat sartu edo hautatu.

Metodo eta gertaera horiek erabiliz, inprimakia bidaltzeko prozesua kontrola dezakegu JavaScript-etik. Adibide honek formularioa bidaltzen du botoi bat sakatzen denean.

Balioestea    

Azpiatalak
  • Akats pertsonalizatuak
  • 'Invalid' gertaera
  • 'ValidityState' objektua

Dagoeneko ikusi dugun bezala, HTMLn formularioak baliozkotzeko hainbat modu daude:

  • Lehenetsitako baliozkotzea eskatzen duten sarrera-eremuak erabil ditzakegu, hala nola email sarrera-eremua.
  • Testu motako eremu erregular bati required atributua erantsiz bete beharreko eremu bihurtuz.
  • Baita tipo bereziak erabiliz ere, pattern gisakoak, baliozkotze eskakizunak pertsonalizatzeko.

Hala ere, baliozkotze-mekanismo gehiago ezarri behar ditugunean, konplexuak, hala nola bi eremu edo gehiago alderatzea edo eragiketa baten emaitza kontrolatzea, Gure aukera bakarra balioztatze-prozesua pertsonalizatzea da, API Formularioak erabiliz.

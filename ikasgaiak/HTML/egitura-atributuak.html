Atributu globalak

id, class

Egiturazko elementu gehienek beren izenetan islatzen den helburu inplizitua duten arren, horrek ez du esan nahi behin bakarrik erabili behar direnik dokumentu berean.

Adibidez, <section> eta <aside> bezalako elementu batzuk askotan erabil daitezke egituraren zati ezberdinak irudikatzeko, eta beste batzuk, <div></div> oraindik ere behin eta berriz inplementatzen dira, atalen barruan edukia bereizteko.

Horregatik, HTMLk elementu bakoitzari identifikatzaile pertsonalizatuak esleitzeko erabil ditzakegun atributu globalak definitzen ditu.

id— Atributu horrek elementu bati identifikatzaile bakarra esleitzeko aukera ematen digu.

class— Atributu horrek identifikatzaile bera esleitzen dio elementu-talde bati.

id— atributuak balio bakarreko elementu independenteak identifikatzen ditu, eta class— atributuaren balioa bikoiztu egin daiteke antzeko ezaugarriak dituzten elementuak lotzeko. Adibidez, elkarren artean bereizi behar ditugun <section> motako bi elementu edo gehiago baditugu, id atributua erants diezaiokegu bakoitzari bere helburuak adierazten dituzten balioekin. 

(...)
<main>
<section id="albisteak">
Artikulu luzeak
</section>
<section id="gertukoalbisteak">
Artikulu laburrak
</section>
<aside>
lehen artikulura esteka
Bigarren artikulura esteka
Hirugarren artikulura esteka
</aside>
</main>
(...)

id atributua duten elementuak identifikatuz. Adibideak bi </section> elementu ditu dokumentuaren atal nagusian, artikuluak luzapenaren arabera bereizteko. Elementu horien edukia desberdina denez, estilo desberdinak eskatzen dituzte eta, beraz, balio desberdinekin identifikatu behar ditugu. Lehenengo elementua </section> "albisteak" balioarekin identifikatu da, eta bigarrena "tokiko albisteak" balioarekin.

Bestalde, antzeko ezaugarriak dituen elementu multzo bat identifikatu behar badugu, class atributua erabil dezakegu.

<div></div>

Hurrengo adibideak <div></div> elementuak dituen atal baten edukia banatzen du. Guztiek antzeko edukia dutenez, estilo berberak partekatuko dituzte eta, beraz, balio berarekin identifikatu beharko genituzke (denak mota berekoak dira).


(...)
<main>
<section id="albisteak">
Artikulu luzeak
</section>
<section id="gertukoalbisteak">
Artikulu laburrak
</section>
<section id="misterioa">
Misterio liburuak
<div class="liburuak">
  Libro: IT, Stephen King</div>
<div class="liburuak">
  Libro: Carrie, Stephen King</div>
<div class="liburuak">
  Libro: El resplandor, Stephen King</div>
<div class="liburuak">
  Libro: Misery, Stephen King</div>
</section>
<aside>
lehen artikulura esteka
Bigarren artikulura esteka
Hirugarren artikulura esteka
</aside>
</main>
(...)

Kodean elementu bat dugu <section id="misterioa"> misterio liburuen eduki nagusia irudikatzeko, baina hainbat banaketa sortu ditugu haren barruan <div> elementuekin, edukia antolatzeko.

Elementu horiek class atributuarekin eta "liburuak" balioarekin identifikatu direnez, "liburuak" klasea erreferentziatuz elementu berriak sartzen edo aldatzen ditugun bakoitzean, elementu horiek guztiek klase horri erantsitako propietateen eragina jasango dute.

GARRANTZITSUA: id eta class atributuen balioak HTML elementu batzuek erabiltzen dituzte beste elementu batzuk identifikatzeko, baita CSS arauek eta JavaScript kodeak ere, dokumentuan elementu espezifikoak sartu eta aldatzeko. Hurrengo kapituluetan adibide praktiko batzuk ikusiko ditugu.

Oinarrizkoa: atributu horiei esleitutako balioak arbitrarioak dira. Nahi duen edozein balio eman diezaieke, baldin eta espazio zuririk ez badu (class atributuak espazio asko erabiltzen ditu elementu berari mota asko esleitzeko). Era berean, zure webgunea nabigatzaile guztiekin bateragarria izateko, letrak eta zenbakiak bakarrik erabili beharko zenituzke, beti letra batekin hasi eta karaktere bereziak saihestu.

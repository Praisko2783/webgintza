Oinarrizko egitura

html dokumentu oro nodo berezi eta espezifiko batekin hasten da gaur egun. Nodo hori dokumentuaren hasieran jartzen da, egiten ari garen dokumentuaren html lengoaiaren bertsioa zehazteko.

Adierazpen hori garrantzitsua da nabigatzaileari esateko nola interpretatu eta renderizatu behar duen orriaren html kodea.

Egia esan, gaur egun nabigatzaile gehienek lehenetsita erabiltzen dute, jartzen ez baduzu ere, baina oraindik nabigatzaileren batek egiten du, beraz, beti jartzea gomendatzen dizut.

Ondoren, < html> etiketa jarri beharko genuke. html etiketa raiz elementua da, eta haren barruan jarriko dugu gure eduki guztia. Edukia bitan banatzen da. Alde batetik head edo burua daukagu eta bestetik body edo gorputza.

  
     <! DOCTYPE html >
        < html >
        < head >
        
        < /head >
        < body >
        
        < /body >
        < /html >
(Irudia: exekutatu kodea eta nabigatzeilean ez da ezer ikusten)

Head gure webgunearen burua da. Bertan metadatuak jarriko ditu. Buruan jarritako ezer ez da errenderizatuko. Bere zeregina gure orrialdeari barne-informazioa ematea da.

Body gure webgunearen gorputza da. Gorputzaren barruan ipiniko dugu gure orrian errenderizatu nahi duguna, hau da, pantailan ikustea nahi duguna. Beraz, < body > eta < /body > artean gure html kode osoa idatziko dugu.

Head

meta=”charset UTF-8”

Metadatu garrantzitsua da charset utf 8, nabigatzaileari zer kodifikazio erabili behar duen adierazten baitio. utf-8k tildeak eta abar kontuan hartzen ditu. Idaz dezagun Oñati.

          
            < ! DOCTYPE html >
            < html >
            < head >
              < meta=”charset UTF-8” />
            < /head >
            < body >
              Oñati
            </ body >
            </ html >
(Irudia: exekutatu kodea eta nabigatzailean Oñati ongi)

Utf-8 ez badugu erabiltzen tildea duten karaktereak, gaizki ikusiko dira. Ikus dezagun:

(Irudia: utf-8 meta kenduta exekutatu kodea eta nabigatzailean Oñati gaizki)
meta name = "viewport" content = "width = device-width, initial-scale = 1.0"

Metadatu horrekin, web-orria hainbat pantaila-tamainatara egokitzeko esaten ari gatzaizkio nabigatzaileari. Metadatu hori gabe, webgunea mugikorrean kargatzean haren zatitxo bat baino ez da ikusten. viewport metadatoaren bitartez, gure orrialdearen zabalera mugikorraren zabalera izango dela ziurtatuko dugu. 

          
            <! DOCTYPE html >
            < html >
            < head >
              < meta=”charset UTF-8” />
              < meta name = "viewport" content = "width = device-width, initial-scale = 1.0" />
            </ head >
            < body >
              Oñati
            </ body >
            </ html >
(Irudia: exekutatu kodea eta mugikor bertikala, mugikor horizontala, ordenagailu-pantaila erakutsi)
title etiketa

title da gure dokumentuak izango duen izenburua eta nabigatzailearen erlaitzean agertuko da. HTML ikastaroa jarriko dugu. 

          
            <! DOCTYPE html >
            < html>
            < head >
                < meta=”charset UTF-8” />
                < meta name = "viewport" content = "width = device-width, initial-scale = 1.0" />
                < title >HTML ikastaroa</ title >
            </ head >
            < body >
              Oñati
            </ body >
            </ html >
(Irudia: exekutatu kodea eta erlaitzean orriaren izenburua seinalatu)

Begira zer agertzen den erlaitzean. Arakatzailearekin irekitzen badugu, hemen daukagu. Izenburu honek egundoko garrantzia dauka. Izan ere, bilatzaileek horri begiratzen baitiote. Title hutsik utziz gero, oso irudi txarra ematen baitu.

Favicon

Kontuan hartu beharreko beste metadatu garrantzitsu bat favicon da.

Favicon-a link motako bat da, eta linka jartzen zaio, eta ez metadatua, azken batean baliabide bati lotzen ari zaiolako. Zer ikono erabili behar duzun esango dizugu. Irudi bat izango da .jpg; adibidez, href atributuan irudiaren helbidea jarriko dugu, gorde egingo dugu, eta orrialdea itxi behar badugu, izenburuaren ezkerraldeko erlaitzean gure ikonoa agertzen dela ikusiko dugu.

          
            <! DOCTYPE html>
              < html>
              < head>
                  < meta=”charset UTF-8”/>
                  < meta name = "viewport" content = "width = device-width, initial-scale = 1.0" />
                  < title>HTML ikastaroa</title>
                  < link rel="icon" href="favicon.ico irudira bidea" type="image/x-icon">
              </ head>
              <body>
                Oñati
              </ body>
              </ html>
(Irudia: exekutatu kodea eta erlaitzean orriaren izenburuaren ondoan iruditxoa seinalatu)
meta name="description" content="Beste txoko bat euskararentzat Gure hizkuntzaren zabaltzearen laguntzeko beste ahalegin bat da Praiskopedia"

Gure orrialdearen deskribapena garrantzitsua da hainbat arrazoirengatik:

Gure edukia deskribatzen duelako, eta deskribapenhori da, hain zuzen ere googlen bilaketa bat egiten duzunean agertzen dena.

Hemen aurkitzen duzuen deskribapena etiketa honetan jartzen duzuena da. Beraz, oso garrantzitsua da deskribapena jartzea.

SEO

SEO (Search Engine Optimization) marketin-estrategia bat da, webguneak optimizatzeko balio duena. Metodo honek prozesu natural eta organikoa eskaintzen du. Horretarako, ez da inolako iragarki-kanpainarik ordaindu behar. Hala ere, horrek ez du esan nahi kostu ekonomikorik edo denborazkorik ez duenik.

Horrekin, gure orrialdearen alternatiba izango litzatekeen kanpoko baliabide bat lotzen ari gara, baina horrek ez du esan nahi beste orri bat jarriko dugunik, ez; gure orriak hainbat hizkuntzatan alternatibak dituela esan nahi dugu. Beraz, kontuan izan hau nahiko interesgarria dela, bestela eduki bikoiztua daukazuela antzeman baitezakezu.

(Irudia: bilatzailean praiskopedia.eus eskatu eta erakusten duena)
link rel="canonical" href="https://praiskopedia.eus/"

Horren bidez, gure guneko orri nagusia zein den adierazten ari gara. Eta hori galdetuko diozue zuen buruari? Izan ere, praiskopedian sartzeko googleri www.praiskopedia.eus idatziz gero, webgunea ireki irekitzen da baina ikusiko duzue birbideratu egin dela eta www-ak kendu dizkiola. Hori zuzena izango litzateke, baina batzuetan interesatzen zaizu orrialde batzuk eduki bikoiztutzat ez hartzea, eta Benetako orria zein den edo nola tratatu behar duen esan behar diozue.

(Irudia: praiskopedia.eus iturburuan canonical erakutsi)
Style etiketa

Estiloak, oro har, aparteko dokumentu batean idaztea da modurik onena. Bada orriaren buruan idazten duenik, baita html-arekin nahasirik ere. Funtzionat funtzionatzen dute baina luzera begira oso modu eskas da estiloak antolatzeko. Esandakoa estiloak aparteko dokumentuan, lotura hau erabiliz:

Estiloen gaia CSS ikastaroan ikusiko dugu. Halere adibidetxo bat erakutsiko dizuet estiloen magia ikus dezazuen. Gorputza gorritu egingo dugu.

          
            <! DOCTYPE html >
              < html >
              < head>
                    < meta=”charset UTF-8” />
                    < meta name = "viewport" content = "width = device-width, initial-scale = 1.0" />
                    < title >HTML ikastaroa</ title >
                    < meta property="og:description" content="Beste txoko bat euskararentzat..." />
                    < link rel="canonical" href="https://praiskopedia.eus/"/>
                    < link rel="icon" href="favicon.ico irudira bidea" type="image/x-icon">
                    < style >
                      body{
                        background:red;
                        }
                    </ style >
              </head>
              <body>
                Oñati
              </body>
        </html>
(Irudia: kodea idatzi eta exekutatu, body gorria, Oñati beltza)

Eta pum! Body gorria.

Ez didazue esango ez dela magikoa!

Baina esandakoa, magia ikusteko ongi dago adibidea baina ez da oso gomendagarria, CSS eta HTML nahastea. Askoz erosoago eta luzera etekin handiagoa ematen du estiloak artxibo ezberdinetan idaztea.

Horretarako daukagu goian aipatu lotura: < link rel='stylesheet' href='estiloak.css' >

Horrela, gure estiloak estiloak.css izeneko eta .css luzapena duen beste artxibo batean idatziko genituzke. Baina hori guztia CSS atalean ikusiko dugu.

Ikus dezakezuenez, metadatu asko daude nabigatzaileentzat, zehazki. Badira web orri jakin baterako, sare sozial baterako, nabigatzaile baterako... balioko dutenak ere.

Ikusi ditugunekin webgune ezin ederrago eta funtzionalagoa egin dezakegu. Beraz aurrera. Hauxe daukagu oraingoz gure webgunearen buruan: 
          
            <! DOCTYPE html >
              < html >
              < head>
                    < meta=”charset UTF-8” />
                    < meta name = "viewport" content = "width = device-width, initial-scale = 1.0" />

                    < link rel="canonical" href="https://praiskopedia.eus/"/>
                    < link rel="icon" href="favicon.ico irudira bidea" type="image/x-icon">
                    < style >
                      body{
                        background:red;
                        }
                    </ style >

Tildeak behar bezala erakusteko agindua, pantaila tamaina ezberdinetara egokitzeko agindua, web-orriaren izen burua bilatzailea jakinaren gainean jartzeko, gure gunearen laburpena nabigatzaileek ipini dezaten aurkitutako guneak erakusterakoan, gure webgune nagusiaren izena zein den esan diogu, gure ezaugarria ikono txiki gisara goiko pestañan agertzeko esan diogu eta gure estiloak izango dituen dokumentuaren helbidea eta esteka. Horrezaz gain css magikoarekin gorputza oso-osorik gorritu egin dugu.
